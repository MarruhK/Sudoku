  • Complete a project which I found difficult to the point where I wouldn't know where to begin.

Obstacles I Faced

Generating a solvable board

Generating the board was something that I found extremely difficult. At first, I had no idea on how to mathematically gerate the board; however, I found a method to obtain a "unique" board. I recorded one valid sudoku board and used this to generate new boards.

Each row, column and box should have unique values ranging from 1-9. Creating three groups, each group being three rows, grants the ability to shuffle around the board (via the three groups), while not breaking any sudoku rules. I used the "Fisher-Yates shuffle" to randomize these groups. Within these groups, I would also shuffle the three rows in it. After this shuffling, the program would shuffle the columns in the exact same way, creating a "unique" board.

Taking on too much at once

When I began this project, I wanted to have multiple functionalities and tried to implement them all at once. This was a big head ache and resulted in many logical errors within my code, resulting in me discarding many lines of code. I've learned that I should keep it simple and begin with the most basic functionalities and once that is complete, I should add the more advanced functionalities.

Reflections

Looking back, I'm happy with the overall project; however, there are still many things I would like to add. I would like to add additional buttons, such as saving (serialization), undo and difficulty. Along with this, the implementation of the red textfields (when an illegal number is placed) is very clunky and messy. This is something I would like to clean up when I revisit this project.
